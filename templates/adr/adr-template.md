## How to use this template

- Pair with the `adr` skill; the skill decides whether an ADR is warranted, this file shapes the record.
- Keep facts, assumptions, and opinions distinct. Use placeholders for unknown facts — do not invent dates, metrics, or consensus.
- Explain rejected options fairly. Record trade-offs; no option is universally best.
- Link supporting material instead of copying it.
- Store ADRs where the project keeps them (commonly `docs/adr/`), numbered in sequence.
